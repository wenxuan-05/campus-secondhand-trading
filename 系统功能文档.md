## 1. 项目概述

### 1.1 系统定位

校园二手交易平台是一个面向高校学生的C2C二手商品交易系统，以**西南财经大学**为首批落地场景。系统支持学生发布闲置商品、求购需求，进行线上线下结合的闭环交易，并提供信用评价体系。

### 1.2 核心业务流程

```
卖家发布商品 → 校园大使审核 → 商品上架 → 买家下单
    → 买家付款(模拟) → 卖家生成取货码 → 买家验证取货码
    → 买家确认收货 → 双方互评 → 信用分更新
```

### 1.3 角色体系

| 角色 | 标识 | 权限说明 |
|------|------|----------|
| 普通用户 | `USER` | 浏览、发布商品、下单、聊天、评价 |
| 校园大使 | `CAMPUS_AMBASSADOR` | 用户权限 + 商品审核 + 推广数据看板 |
| 管理员 | `ADMIN` | 全部权限：用户管理、商品管理、订单管理、退款仲裁、大使审核 |

## 4. 安全体系

### 4.1 JWT认证流程

```
注册/登录 → 服务端生成JWT Token → 客户端存储 → 每次请求携带Bearer Token
    → JwtAuthenticationFilter拦截校验 → 解析userId/role/studentId
    → 校验tokenVersion（密码重置后+1使旧Token失效）
    → 同步数据库最新角色 → 存入UserContext(ThreadLocal) → 放行
```

**JWT令牌包含字段**: sub(userId)、studentId、email、role、tokenVersion、iat、exp

### 4.2 角色权限控制

**注解方式**: `@RequireRole` + `RoleInterceptor` 拦截器

- 类级注解：`@RequireRole({"ADMIN", "CAMPUS_AMBASSADOR"})` 整个Controller需要这些角色
- 方法级注解：`@RequireRole("ADMIN")` 特定方法需要更严格权限
- 未标注 = 任何已登录用户可访问

### 4.3 公开端点（无需认证）

| 路径 | 方法 | 说明 |
|------|------|------|
| `/api/user/login` / `/api/user/register` | POST | 登录注册 |
| `/api/user/send-code` / `/api/user/reset-password` | POST | 验证码/重置密码 |
| `/api/goods/{id}` | GET | 商品详情 |
| `/api/goods/search` | GET | 商品搜索 |
| `/api/user/{id}` | GET | 用户公开信息 |
| `/api/file/**` | GET | 文件代理 |
| `/ws/chat` | WS | WebSocket（Token认证） |

### 4.4 密码安全

- 密码存储：MD5哈希（`DigestUtils.md5DigestAsHex`）
- 密码重置后 `tokenVersion + 1`，所有旧JWT Token即时失效
- 登录时校验Token版本号与数据库一致性

### 4.5 跨域配置 (CORS)

允许所有来源、GET/POST/PUT/DELETE/OPTIONS方法、支持凭证传递、maxAge=3600s。

## 5. 用户模块

### 5.1 注册 `POST /api/user/register`

1. 校验密码一致性
2. 校验学号为8位数字
3. 检查学号唯一性
4. 校验邮箱验证码（Redis存储，5分钟有效）
5. 自动生成学校邮箱：`{studentId}@smail.swufe.edu.cn`
6. 密码MD5加密，初始化：信用分=80，角色=USER，状态=1(正常)

### 5.2 登录 `POST /api/user/login`

1. 支持学号或邮箱登录（自动判断是否含`@`）
2. 密码MD5校验
3. 检查账号状态：status=0且封禁到期→自动解封（恢复60分），未到期→提示到期时间
4. `admin`学号强制设为ADMIN角色
5. 生成JWT Token，返回Token+用户信息

### 5.3 发送验证码 `POST /api/user/send-code`

- type=register：检查学号未注册
- type=reset：检查学号已注册
- 生成6位随机验证码，5分钟有效，通过QQ邮箱发送

### 5.4 重置密码 `POST /api/user/reset-password`

校验学号+验证码 → 新密码MD5加密 → **tokenVersion+1**（强制所有设备重新登录）

### 5.5 个人资料 `GET/PUT /api/user/profile`

支持修改：昵称、头像、手机号、学校名称。

### 5.6 信用分系统

**信用等级**:
| 分数 | 等级 |
|------|------|
| ≥90 | 优秀 |
| 70-89 | 良好 |
| 50-69 | 一般 |
| 30-49 | 较差 |
| <30 | 极差 |

**变化规则**:
| 事件 | 变化 |
|------|------|
| 好评（4-5星） | +2 |
| 差评（1-2星） | -5 |
| 中评（3星） | 不变 |
| 带图评价 | 额外+1 |
| 信用分<60 | 自动封禁7天 |
| 封禁到期 | 自动恢复60分 |

## 6. 商品模块

### 6.1 发布商品 `POST /api/goods`

**发布限制**:
- 正常用户(status=1)：每日10条
- 限制用户(status=2)：每日3条
- 在售商品上限：50件
- 初始状态=1（审核中），需校园大使审核后才能上架
- 自动设置30天过期时间

### 6.2 商品操作

| 操作 | 接口 | 关键逻辑 |
|------|------|----------|
| 编辑 | `PUT /api/goods/{id}` | 已售/已删除不可编辑 |
| 下架 | `PUT /api/goods/{id}/off` | 仅在售→下架 |
| 重新上架 | `PUT /api/goods/{id}/on` | 仅下架→审核中，每日限3次刷新 |
| 刷新 | `PUT /api/goods/{id}/refresh` | 在售商品，每日限3次，更新时间排序权重 |
| 删除 | `DELETE /api/goods/{id}` | 逻辑删除(status=6)，有进行中订单不可删除 |

### 6.3 商品搜索 `GET /api/goods/search`

**搜索条件**: 关键词、分类、价格区间、成色、可议价/快递、排序（时间/价格/热度/位置）
**亮点**: 根据当前用户宿舍楼自动进行**位置优先排序**

### 6.4 商品详情 `GET /api/goods/{id}`

每次访问自动增加浏览量(viewCount+1)。

## 7. 订单模块

### 7.1 创建订单 `POST /api/order`

- 校验商品在售且不能买自己的商品
- **商品状态即刻变为3（已售）**，杜绝超卖
- 订单号格式：`CS` + yyyyMMddHHmmss + 4位随机数
- 初始状态=10（待付款）

### 7.2 订单操作流程

| 操作 | 接口 | 状态变化 | 说明 |
|------|------|----------|------|
| 支付(模拟) | `PUT /api/order/{id}/pay` | 10→20 | 记录支付时间 |
| 生成取货码 | `PUT /api/order/{id}/pickup-code` | — | 卖家操作，6位数字，过滤弱密码 |
| 验证取货码 | `POST /api/order/{id}/verify-pickup` | 20→30 | 买家输入验证码，校验匹配 |
| 确认收货 | `PUT /api/order/{id}/confirm` | 30→40 | 记录确认时间 |
| 取消订单 | `PUT /api/order/{id}/cancel` | →90 | 买家或卖家均可，**恢复商品在售** |

### 7.3 取货码安全机制

- 禁止全部相同数字（如111111）
- 禁止连续数字序列（如123456）
- 使用`SecureRandom`生成

### 7.4 订单查询

- `GET /api/order/buyer` — 买家订单列表（分页）
- `GET /api/order/seller` — 卖家订单列表（分页）

## 8. 评价模块

### 8.1 提交评价 `POST /api/reviews`

1. 订单状态必须=40（待评价）
2. 自动判断评价方向：买家评卖家(type=1) 或 卖家评买家(type=2)
3. 防重复评价
4. 评价内容：评分(1-5)、标签(JSON)、文字(0-200字)、晒图(JSON)
5. **更新被评价人信用分**（按5.6节规则）
6. 更新交易笔数和好评率
7. 双方都评价后 → 订单状态=50（已评价）

### 8.2 评价查询

- `GET /api/reviews/order/{orderId}` — 某订单所有评价
- `GET /api/reviews/user/{userId}` — 某用户收到的评价（最近10条）

### 8.3 自动好评（定时任务）

每日凌晨2点：确认收货>7天未评价的订单，自动生成5星好评（标记is_auto=1）。

## 10. 即时通讯模块

### 10.1 WebSocket实时通信

**连接**: `ws://host:port/ws/chat?token={jwt_token}`
**认证**: `WebSocketAuthInterceptor` 从URL参数提取JWT Token进行校验

### 10.2 消息类型

| type | 含义 | 处理 |
|------|------|------|
| 1 | 文本 | **敏感词过滤**后存储和转发 |
| 2 | 图片 | URL直接存储 |
| 3 | 商品卡片 | 根据goodsId查询商品信息生成卡片JSON |
| 4 | 出价 | 包含出价金额和状态 |

### 10.3 消息处理流程

```
客户端发送 → 类型特定处理（敏感词过滤/商品卡片/出价数据）
    → 持久化到chat_messages
    → 更新chat_sessions（最后消息+时间）
    → 发送给在线接收者（如果在同一房间）
    → 回显给发送者
    → 敏感词过滤通知（系统消息）
```

### 10.4 会话管理

- 基于商品聊天：`goods_{goodsId}_{uid1}_{uid2}` (WebSocket自动创建)
- 基于求购聊天：`buy_{requestId}_{uid1}_{uid2}` (通过API创建)
- 会话列表合并新旧两种格式
- 每个会话返回：对方信息、最后消息、未读数、关联商品信息

### 10.5 聊天API

| 接口 | 说明 |
|------|------|

## 12. 退款模块

### 12.1 退款申请 `POST /api/refunds`

- 仅买家可申请
- 订单状态必须为20(待面交)或30(待确认)
- 防重复申请
- 订单状态→60（退款中）

### 12.2 卖家处理

| 操作 | 结果 |
|------|------|
| 同意退款 | 退款完成，订单→80，**商品恢复在售** |
| 拒绝退款 | 订单→70（退款纠纷） |

### 12.3 自动仲裁升级（定时任务）

卖家48小时未处理的退款 → 自动升级为平台仲裁（状态:1→4）。

### 12.4 平台仲裁 `POST /api/refunds/{id}/arbitrate`

| decision | 结果 |
|----------|------|
| 1（支持买家） | 退款完成，商品恢复在售 |
| 0（支持卖家） | 退款驳回，订单恢复→30(待确认) |

## 14. 校园大使系统

### 14.1 申请流程

```
用户提交申请 → 管理员审核 → 通过：角色→CAMPUS_AMBASSADOR，分配工号
                          → 驳回：通知原因（默认友好话术）
```

**申请表单**: 姓名、学号、手机号、学院/专业、宿舍楼栋（29个选项）、社群资源类型（dormitory/department/club/other）、申请理由（10-200字）
**防重复**: 同一学号已有待审核申请时不允许重复提交

### 14.2 商品审核（校园大使核心职责）

| `GET /ambassador/goods/pending` | 待审核商品（最早提交优先） |
| `PUT /ambassador/goods/{id}/approve` | 审核通过→status=2，记审核人，设30天过期 |
| `PUT /ambassador/goods/{id}/reject` | 审核驳回→status=5，需填写驳回理由 |
| `GET /ambassador/goods/reviewed` | 我的审核记录 |

### 14.3 推广数据看板

- 累计审核/通过/驳回/待审数量
- 审核通过率（保留2位小数）
- 今日审核/通过/驳回数据

```

    → requireAdmin标记 → 仅ADMIN
    → requireAmbassador标记 → AMBASSADOR或ADMIN
    → 失败重定向到首页/dashboard
```

- 请求拦截：自动附加 `Authorization: Bearer {token}`
- 响应拦截：code≠200弹错误提示；HTTP 401自动登出跳转到首页

开发端口3000 → `/api` 代理到 `http://localhost:8088`，`/ws` WebSocket代理

### 18.6 设计风格
